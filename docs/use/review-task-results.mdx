# Review Task Results

Review a task before you decide what to do with its output.

Task detail pages can show task status, messages, streamed logs, terminal output, an implementation summary, checkpoints, diffs, and Git operations.

## Before You Begin

You need access to the task.

## Steps

1. Open **Tasks**.

2. Select the task you want to review.

3. Read the task messages and result summary.

   For Build tasks, open **Implementation Summary** to review the latest summary of the work. When a Build task succeeds, Verft opens the latest **Implementation Summary** automatically. If you are already viewing the task while it finishes, the summary expands without reloading the page.

4. Check the task status to see whether it is still queued, running, awaiting review, answered, completed, failed, or archived.

5. For Build tasks, inspect the proposed file changes before applying or rejecting them.

6. For Ask tasks, review the answer and any supporting context the agent provided.

7. Check logs if you need to understand what happened during execution.

## Result

You know whether the task is still working, needs review, produced an answer, proposed file changes, or failed.

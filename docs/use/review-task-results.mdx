# Review Task Results

Review a task before you decide what to do with its output.

Task detail pages can show task status, messages, streamed logs, terminal output, checkpoints, diffs, and Git operations.

## Before You Begin

You need access to the task.

## Steps

1. Open **Tasks**.

2. Select the task you want to review.

3. Read the task messages and result summary.

4. If the task shows result section tabs, start with **Implementation Summary** for a Build task or **Summary** for an Ask task.

5. Use **Timeline** if you need to see detailed run events.

6. Check the task status to see whether it is still queued, running, awaiting review, answered, completed, failed, or archived.

7. For Build tasks, inspect the proposed file changes before applying or rejecting them.

8. For Ask tasks, review the answer and any supporting context the agent provided.

9. Check logs if you need to understand what happened during execution.

## Result

You know whether the task is still working, needs review, produced an answer, proposed file changes, or failed.

## What to Expect

For Build task runs, **Implementation Summary** appears before **Timeline** in the result section tabs. **Diff** appears when there are proposed file changes.

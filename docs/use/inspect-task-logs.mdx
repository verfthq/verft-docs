# Inspect a Task Timeline

Use the task timeline when you need to understand what happened during a Verft task run.

This page covers the timeline inside a task. For Docker container logs on the Verft server, see [Inspect Verft server logs](/inspect-logs).

## Before You Begin

You need access to the task.

## Steps

1. Open **Tasks**.

2. Select the task you want to inspect.

3. Review the task messages and status first.

4. In the task history, open **Timeline** for the run you want to inspect.

5. Check the most recent entries for failures, queued work, provider output, or terminal activity.

6. If **Raw JSON** is available, select it to download the raw provider JSONL for that run.

## Result

You can see task-level execution events from the browser and use them to understand task progress or failures.

## Troubleshooting

- Problem: **Raw JSON** is not shown.
  - Solution: Use the timeline entries and task messages. Raw provider JSON is only available for some runs.

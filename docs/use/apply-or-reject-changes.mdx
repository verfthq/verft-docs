# Apply or Reject Changes

Decide what happens to a Build task's proposed file changes after you review them.

Build tasks produce reviewable output. They do not need to move forward automatically.

## Before You Begin

You need:

- access to the task
- permission to edit tasks
- a Build task with proposed changes or a pending checkpoint

## Steps

1. Open **Tasks**.

2. Select the Build task.

3. Review the proposed changes, changed files, and diff.

4. Choose the action that matches your decision:

   - Apply the changes if you want to keep them in the task workspace.
   - Reject the changes if you do not want to keep that proposal.
   - Revert applied changes if you need to undo a previously applied proposal.
   - Push or merge only after you are ready to move changes through Git.

5. Confirm the action if Verft asks for confirmation.

## Result

The task updates to reflect the review action you selected.

## What to Expect

Checkpoint actions can be unavailable while a task is queued or actively running.

# Configure Provider Credentials

Configure credentials so Verft can access GitHub and AI providers.

Credentials are configured in **Settings**, not in `.env`.

For Codex and Claude Code, you can either add provider API keys or sign in with a Codex or Claude subscription account from Verft.

## Before You Begin

You need:

- access to **Settings**
- permission to edit settings
- the credential values you want to add, or access to the Codex or Claude account you want to sign in with

## Steps

1. Open **Settings**.

2. Open **Credentials**.

3. Add the credentials you need:

   - **GitHub Token** for authenticated GitHub access.
   - **OpenAI Key** for Codex and OpenAI-backed features.
   - **Anthropic Key** for Claude Code.

4. Select **Save Credentials**.

5. Optional: sign in with a provider account instead of using only API keys:

   - Select **Sign in Codex** to run `codex login --device-auth` in a terminal window.
   - Select **Sign in Claude** to run `claude auth login` in a terminal window.
   - Select **Open Provider Setup Terminal** when you need a shared setup terminal for Codex and Claude files.

6. Optional: open **Codex** or **Claude Code** to configure default model, model list, effort, or base URL overrides.

## Result

The credential status tags in **Settings** show whether each credential is configured or missing.

## What to Expect

Credentials are write-only. Verft can show whether a credential is configured, but it does not show the stored secret value.

Provider login files created through the setup terminals are stored in Verft's shared base volume so new tasks can reuse them.
